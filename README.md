# Scientific Engine (C++ / OpenGL 4.6)

> A real-time computational physics and procedural graphics engine built from scratch.

Hi, I'm Ayodeji. I built this engine because I am fascinated by what happens when you combine raw mathematics, computational physics, and low-level memory management. 

Rather than using an off-the-shelf engine like Unity or Unreal, I wanted to build the entire pipeline from the ground up—from compiling GLSL shaders and managing the GPU's memory buffers, to implementing mathematical noise algorithms and building a custom rendering loop. 

This repository serves as my ongoing laboratory for exploring complex simulations, graphics programming, and high-performance C++ architecture.

## 🌌 Current State: Procedural Planetary Generation
The foundation of the engine is complete, featuring a robust 3D rendering pipeline and a highly optimized procedural generation system.

* **Algorithmic Mesh Generation:** Planets are generated by mathematically subdividing an Icosahedron to ensure uniform vertex distribution, bypassing the UV distortion issues of standard sphere generation.
* **Procedural Topography:** Mountain ranges and ocean basins are sculpted in real-time using Ridged Multifractal noise, Domain Warping, and 3D Simplex noise. 
* **Elevation-Based Shading:** A custom Fragment Shader dynamically paints the terrain biomes (sand, grass, rock, snow) based strictly on vertex distance from the planet's core.
* **Interactive Illumination:** Lambertian reflectance models with dynamic, ray-casted point lights that can be physically dragged through the 3D space.
* **Laboratory Dashboard:** Integrated **Dear ImGui** to provide real-time hardware telemetry (FPS, frame-times), memory profiling (poly-counts), and live parameter tuning.

## The Roadmap: Computational Physics

With the architectural foundation (UI, Rendering, and Camera math) locked in, the engine is currently being expanded into a full simulation sandbox.

### Phase 1: Smoothed Particle Hydrodynamics (SPH)
*Currently in active development.*
Implementing a real-time hydrostatic fluid simulation to interact with the procedural planetary surfaces. This involves building spatial hashing grids (or Octrees) to manage thousands of fluid particles, calculating pressure gradients, and simulating viscosity on the GPU.

### Phase 2: N-Body Gravity & Orbital Mechanics
Replacing static worlds with a dynamic solar system. This phase will focus on numerical integration (like Runge-Kutta or Verlet integration) to simulate the gravitational pull between massive bodies in real-time without system energy loss.

### Phase 3: General Relativity Visualizations
Pushing the engine beyond classical mechanics to visualize spacetime curvature and relativistic effects using non-Euclidean rendering techniques.

## 🛠️ Architecture & Tech Stack

This engine is built with a strict emphasis on performance, clean memory management, and modern C++ practices.

* **Language:** C++20
* **Graphics API:** OpenGL 4.6 (Core Profile)
* **Window & Hardware:** SDL2
* **Mathematics:** GLM (OpenGL Mathematics)
* **Interface:** Dear ImGui (Docking Branch)
* **Build System:** CMake + vcpkg
